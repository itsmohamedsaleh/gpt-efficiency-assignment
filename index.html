Introduction

This project, titled "ChatGPT Validator," aims to systematically evaluate the performance and efficiency of ChatGPT across various domains. By developing an interactive platform, we provide a comprehensive analysis of ChatGPT's capabilities, strengths, and areas for improvement.

The platform is designed to achieve multiple objectives:

  • Demonstrate the integration of frontend and backend technologies in creating a data-driven web application.
  • Showcase the process of connecting and managing databases using modern development tools.
  • Evaluate ChatGPT's responses against a standardized set of questions to assess accuracy and response times.
  • Present the findings through intuitive visualizations, enabling users to understand ChatGPT's performance metrics effectively.

By leveraging data from diverse fields such as computer security, history, and social science, this project provides valuable insights into how ChatGPT operates within different contextual frameworks. The results aim to inform users about the practical applications and limitations of using ChatGPT in various professional and academic settings.

Research Approach

The research methodology for the "ChatGPT Validator" project encompasses several key steps, ensuring a robust and comprehensive evaluation of ChatGPT's performance. The approach integrates both development practices and analytical techniques as outlined below:

1. Environment Setup

Utilizing Visual Studio Code (VSCode) as the primary development environment, we established the necessary tools and frameworks required for both frontend and backend development. The terminal was employed to initialize and manage the project repository, facilitating version control and dependency management.

2. Database Creation and Management

We created a MongoDB database through the terminal, setting up the required collections to store questions, answers, and evaluation metrics. The database schema was designed to accommodate data from three CSV files, each representing different domains: computer security, history, and social science.

3. Backend Development

Using Node.js and Express.js, we developed the server-side application to handle data operations and API integrations. Express.js served as the framework to manage routing and middleware, ensuring efficient communication between the frontend and the MongoDB database.

4. Integration with ChatGPT API

To connect with ChatGPT, we utilized the chat.completion endpoint of the OpenAI API. This integration enabled automated querying of ChatGPT with the questions extracted from the CSV files. The responses generated by ChatGPT were then captured and stored in the MongoDB database for further analysis.

5. Data Population

We uploaded three CSV files containing a comprehensive set of questions and their correct answers into the database. Subsequently, ChatGPT's responses to these questions were also populated into the database, allowing for a direct comparison between human-provided answers and AI-generated answers.

6. Comparative Analysis

The core of the research involved comparing ChatGPT's responses against the correct answers to evaluate accuracy and efficiency. Metrics such as response time and correctness percentage were calculated to quantify performance. This comparative analysis provided insights into ChatGPT's reliability and effectiveness across different subject domains.

7. Results Visualization

Finally, the results of the evaluations were presented through interactive charts and statistics on the platform. This visualization aids in understanding the data-driven outcomes, highlighting areas where ChatGPT excels and identifying potential areas for improvement.
