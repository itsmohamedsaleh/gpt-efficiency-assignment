Introduction

This project, titled ChatGPT Validator, aims to systematically evaluate the performance and efficiency of ChatGPT across various domains. By leveraging a structured approach that integrates data management, web development, and artificial intelligence, the project provides insightful comparisons between human-generated answers and those produced by ChatGPT.

The workflow encompasses several key phases:

  1. Database Creation: Utilizing the terminal and Visual Studio Code (VSCode), we initiated the creation of a MongoDB database to store our dataset.
  2. Backend Development: Employing Node.js and Express, we established a robust connection to the MongoDB database, facilitating seamless data operations.
  3. API Integration: Through the OpenAI ChatGPT API, we implemented functionalities to generate responses based on user queries.
  4. Data Population: We uploaded three CSV files containing a comprehensive set of questions and answers, which were then imported into the database. Subsequently, ChatGPT's responses to these questions were also stored for comparative analysis.
  5. Performance Evaluation: By comparing the accuracy and response times of ChatGPT against the provided answers, we assessed the model's efficiency and reliability.
  6. Results Presentation: The findings were visualized through various charts and statistical data, offering a clear overview of ChatGPT's performance across different domains.

The integration of these components not only demonstrates the practical applications of ChatGPT but also provides a framework for evaluating AI performance in real-world scenarios.

Research Approach

The research methodology for the ChatGPT Validator project was meticulously designed to ensure comprehensive evaluation and reliable results. The approach can be broken down into the following detailed steps:

  • Environment Setup: Using Visual Studio Code (VSCode) as the primary development environment, we initialized the project repository and configured necessary dependencies.
  • Database Creation: Through the terminal, we set up a MongoDB database to store all relevant data. This involved defining schemas for questions, answers, and ChatGPT responses to maintain structured and efficient data storage.
  • Backend Development: Leveraging Node.js and Express, we developed the server-side application to handle API requests, manage database interactions, and facilitate communication between the frontend and the backend systems.
  • API Integration: We integrated the OpenAI ChatGPT API using the `chat.completion` endpoint. This enabled the application to send queries to ChatGPT and retrieve generated responses programmatically.
  • Data Ingestion: Three CSV files containing a diverse set of questions and corresponding correct answers were uploaded and parsed. These data points were then inserted into the MongoDB database, ensuring that each entry was properly indexed and retrievable.
  • Populating ChatGPT Responses: For each question in the database, a request was sent to the ChatGPT API to obtain its answer. These responses were stored in the database alongside the original questions and correct answers.
  • Comparative Analysis: We conducted a thorough comparison between the correct answers and ChatGPT's responses. Metrics such as accuracy, response time, and efficiency were calculated to evaluate performance.
  • Result Visualization: The analyzed data was presented through interactive charts and statistical summaries, providing clear insights into ChatGPT's strengths and areas for improvement across different domains.

By following this structured approach, the project ensures a systematic evaluation of ChatGPT's capabilities, offering valuable data-driven insights into its performance.
